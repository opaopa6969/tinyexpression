# TinyExpression LSP 要求仕様（診断とQuick Fix）

更新日: 2026-02-27  
対象: `tools/calculator-lsp-vscode` (`CalculatorLanguageServer`)

## 1. この仕様の位置づけ

この文書は、実装都合ではなく**ユーザー要求ベース**で、次を固定する。

- どの入力でどの診断コードを出すか
- どの診断でQuick Fixを出すか
- Quick Fixが何を編集するか

## 2. 要求（優先順位）

1. 可能な限り `TE009`（不要トークン）を出さない。  
2. 構造エラーは具体コードを優先する。  
3. 診断とQuick Fixはペアで提供する（出せるものは必ず出す）。  
4. CRLF (`\r\n`) でも診断コードがブレない。  

## 3. エラー判定ルール（要求からの決定）

### 3.1 `if (...)` / `else` の `{` 欠落

- 期待コード: `TE010`
- 期待メッセージ: `構文の並びが想定と一致しません` + detail `ブロック開始 '{' が必要です`
- Quick Fix:
  - `if(...)` または `else` の直後へ `{` を挿入

### 3.2 `match` のケース区切り `,` 欠落

- 例:  
  `$name=='肉太郎' -> 1` の次行に `$didCheck -> 2,` が続く
- 期待コード: `TE013`
- 期待メッセージ: `match 構文が不正です` + detail `match case の区切りに ',' が必要です`
- Quick Fix:
  - 次ケース先頭位置に `,` を挿入

### 3.3 `match` の `default` 前 `,` 欠落

- 期待コード: `TE014`
- Quick Fix: なし（現状）

### 3.4 `->` 右辺式欠落（`-> ,` / `-> }` / `-> EOF`）

- 期待コード: `TE013`
- Quick Fix:
  - `-> 0` を補完

### 3.5 `var` 行の `;` 欠落

- 期待コード: `TE006`
- Quick Fix:
  - 診断位置ではなく**宣言行末**へ `;` を挿入

### 3.6 `if` 条件括弧の未閉鎖

- 期待コード: `TE004`（外部メッセージからの補完含む）
- Quick Fix:
  - 未閉鎖 `(` の行末へ `)` を挿入

### 3.7 `set if not exists` の `if`

- 要求:
  - 制御構文 `if (...)` として誤認しない
- 実装契約:
  - `if` は `if` + `(` のときのみ制御構文扱い

## 5. 現行の非対応/限定対応

- `TE002`, `TE011`, `TE014`, `TE015`, `TE020` は自動修正なし（安全側）
- `TE009` は最終フォールバックとして残すが、要求上は削減対象

## 6. 受け入れ基準（テスト観点）

1. `if(...)` 後に `{` がない式は `TE010` になり、Quick Fix候補が1件以上出る。  
2. `match` ケース間の `,` 欠落は `TE013` になる。  
3. `match` `->` 右辺欠落は `TE013` になる。  
4. `var` の `set if not exists` を含む式で `TE017` 誤検出が出ない。  
5. `TE009` は上記に該当しないときのみ出る。
